### T9 — 피드백 반영 (피드백.md 기반)

**ThemeContext (src/context/ThemeContext.tsx)**
- `{ dark, setDark }` React Context 전역 제공
- `main.tsx`에서 `<ThemeProvider>`로 앱 전체 감싸기
- 모든 컴포넌트에서 `useTheme()` 훅으로 접근

**다크모드 적용 범위:**
- Sidebar: 배경, 텍스트, hover, 설정 패널
- ChatMessage: AI 텍스트 색상, Thinking 패널 (`#0f2744` / `#f0f9ff`), 코드블록 (`oneDark`/`oneLight`)
- ChatInput: 박스 배경, 테두리, 텍스트, placeholder
- WelcomeScreen: 그리팅 텍스트, 카드

**Thinking 토글 (채팅 입력창 우측)**
- Brain 아이콘 + "Thinking" 텍스트 버튼 형태
- ON: 파란 배경 + 테두리 / OFF: 투명
- `params.thinking` 상태와 연동

**설정 패널 개편 (Sidebar 하단)**
- 접힌 상태: 톱니바퀴 아이콘 → 고정 팝업 모달
- 펼친 상태: 다크모드 토글 + Sampling + Temperature 바로 노출
- Thinking 항목은 사이드바에서 제거 (채팅 입력창으로 이동)

**이미지 분석 태스크 추가 (types.ts)**
- `id: 'image'`, `/test.jpg` 자동 첨부
- 프롬프트: "참가기업 알려줘"
- 이미지 태스크 선택 시 `pendingImage` 상태로 자동 첨부

**Thinking 프롬프트 강제 주입 (api/chat.ts)**
- `effectiveThinking === true` + 마지막 user 메시지에 `\n\n질문의 의도를 파악해` 추가
- 이미지 포함 시 `effectiveThinking = false` 강제

**Sampling 로직 (api/chat.ts)**
- `sampling: false` → `temperature: 0` (greedy decoding)
- 이미지/문서 포함 시 temperature 파라미터 제외

**빈 응답 content 처리 (useChat.ts)**
- 스트림 종료 시 `!m.content && m.thinking` → thinking을 content로 승격
- thinking=false이지만 `reasoning_content`만 나오는 경우 대비

### T11 — API 로그 패널 (App.tsx 하단)

**구조:**
- 화면 최하단 고정 32px 헤더 ("API LOG" + 로그 개수)
- 클릭 시 최대 320px까지 확장 (CSS max-height 트랜지션)
- 왼쪽 260px: 엔트리 목록 (type + 시각 + 미리보기)
- 오른쪽: 선택 항목 전체 내용 (`<pre>` 포맷)
- clear 버튼: 로그 전체 삭제

**로그 타입:**
| 타입 | 색상 | 발생 시점 |
|------|------|----------|
| REQUEST | 파랑 (#38bdf8) | fetch 직전 (base64 이미지 생략) |
| RESPONSE | 초록 (#4ade80) | HTTP 200 OK 수신 |
| ERROR | 빨강 (#f87171) | HTTP 오류 또는 context too long |
| DONE | 보라 (#a78bfa) | SSE [DONE] 수신 |

**구현 위치:**
- `api/chat.ts`: `onLog?: (entry: ApiLogEntry) => void` 파라미터 추가
- `hooks/useChat.ts`: `apiLogs` state + `clearApiLogs` 콜백 노출
- `types.ts`: `ApiLogEntry { id, time, type, data }` 타입 추가
- `App.tsx`: 로그 패널 UI + `logOpen`, `selectedLog` state
