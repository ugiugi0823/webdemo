### T4 — 텍스트 채팅 API 연동

**파일:** `src/api/chat.ts`, `src/hooks/useChat.ts`

- **엔드포인트:** `POST /api/vllm/v1/chat/completions`
- **인증:** `Authorization: Bearer <key>` (키: `vllm_api.key` 참고)
- **스트리밍:** `stream: true`, SSE `data:` 라인 파싱
- **멀티턴:** 전체 대화 히스토리를 `messages[]` 배열로 누적 전송
- **중단:** AbortController로 스트리밍 중단 지원
- **에러:** API 오류 시 메시지 버블에 에러 텍스트 표시

**요청 구조:**
```json
{
  "model": "LLM42",
  "messages": [...],
  "stream": true,
  "chat_template_kwargs": { "thinking": true/false }
}
```

---

### T5 — Thinking 토글 + 파라미터

**사이드바 하단 배치:**
- **Thinking 토글:** `chat_template_kwargs.thinking` true/false
- **Sampling 토글:** 활성 시 temperature 적용
- **Temperature 슬라이더:** 0.0 ~ 2.0, 기본 1.0, 그라디언트 트랙

**응답 파싱:**
- vLLM `delta.reasoning_content` → Thinking 버퍼로 분리
- 또는 응답 내 `<think>...</think>` 태그 파싱 (fallback)
- Thinking 섹션: 뇌 아이콘 + "추론 과정" 버튼으로 접기/펼치기

### T6 — 태스크 사이드바

**5개 태스크 (기존 기능 유지):**

| 태스크 | 시스템 프롬프트 | 예제 프롬프트 |
|--------|----------------|---------------|
| Free (Multi turn dialogue) | 없음 | 없음 |
| Question Answering | QA 특화 시스템 프롬프트 | `[내용]: ... [질문]: ...` |
| Keyword Extraction | 키워드 추출 특화 | `[텍스트]: ...` |
| Summary | 요약 특화 | `[내용]: ...` |
| Query Complete | 쿼리 완성 특화 | `[쿼리]: ...` |

- 선택 시 입력창에 예제 프롬프트 자동 로드
- 시스템 프롬프트는 API 요청 시 `role: "system"` 메시지로 주입
- 현재 선택 태스크 보라색 하이라이트
- topbar에 현재 태스크명 + "활성" 배지 표시

### T7 — 이미지 업로드 및 추론

**파일:** `src/components/ChatInput.tsx`, `src/api/chat.ts`

- **첨부 방법:** 📎 버튼 클릭 또는 드래그 앤 드롭
- **지원 형식:** JPEG, PNG, WebP, GIF (최대 10MB)
- **미리보기:** 썸네일 + 파일명 + "이미지 추론 모드" 표시
- **제거:** X 버튼으로 첨부 취소

**API 전송 포맷 (`이미지추론.md` 기반):**
```json
{
  "role": "user",
  "content": [
    {
      "type": "image_url",
      "image_url": { "url": "data:{mimeType};base64,{base64}" }
    },
    {
      "type": "text",
      "text": "<|mime_start|>{\"id\":\"image_00\",\"type\":\"{mimeType}\",\"filename\":\"{name}\"}<|mime_end|><|image_start|><|IMAGE_PAD|><|image_end|>\n{사용자 질문}"
    }
  ]
}
```

### T8 — Docker 배포

**파일:** `prod/Dockerfile`, `prod/nginx.conf`, `prod/docker-compose.yml`

**배포 명령 (WEBDEMO 루트에서):**
```bash
docker compose -f prod/docker-compose.yml up -d
# → http://localhost:8080
```

**Dockerfile (multi-stage):**
1. `node:20-alpine` — `npm ci` + `npm run build`
2. `nginx:alpine` — 빌드 결과물 + nginx.conf 복사

**nginx.conf 주요 설정:**
- SPA fallback (`try_files $uri /index.html`)
- `/api/vllm/` → `http://192.168.0.234:9015` 역방향 프록시
- `/api/fastapi/` → `http://192.168.0.234:9016` 역방향 프록시
- SSE 스트리밍: `proxy_buffering off`, `proxy_read_timeout 300s`
- 정적 파일 1년 캐싱
